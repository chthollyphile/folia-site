# 播放、歌词与视觉模式

Folia 的核心体验在播放页。这里的重点不是传统播放器那种“按钮很多”，而是让歌词、封面和节奏成为主角。

## 播放页的核心元素

![alt text](../public/screenshots/guide/playback-and-lyrics/image.png)

- 当前歌曲信息：标题、歌手、专辑、封面
- 歌词区域：按时间轴滚动或逐字展示
- 底部控制：播放、暂停、切歌、进度、音量
- 右侧面板：循环、视觉模式、主题、AI 主题、歌词来源等

## 内置视觉模式

当前内置模式包括：

- `流光`：默认模式，适合大多数歌曲
- `心象`：更强调氛围和文字结构
- `云阶`：更偏排版层次与段落推进
- `浮名`：更强调几何背景和镜头感
- `群唱`：更具角色感与表情化的歌词演出
- `倾诉`：提供逐字呼吸和旋转效果
- `莫奈`：支持封面或自定义右侧肖像图片、副歌特效，并具备多种背景排版布局（全屏叠色、半屏渐变）以及关键字着色选项。

不同模式共享同一份播放数据，但会用不同的排版与动画逻辑来渲染。

## 循环、收藏与音量

右侧控制面板提供：

- 循环模式：关闭 / 列表循环 / 单曲循环
- 收藏按钮
- 音量滑杆与静音
- 动画强度：`calm` / `normal` / `chaotic`

## 背景与主题

Folia 支持四类主题来源/背景：

- 默认主题
- AI 主题
- 自定义主题
- 网页背景（URL Background）：可以在设置中添加外部网页 URL 作为 iframe 嵌入在播放页最底层，丰富背景的视觉体验。

另外还能切换：

- 明亮 / 暗色外观
- 是否叠加封面主色到背景

如果开启“歌曲主题自动切换”，切歌时会优先恢复该歌曲上次使用的主题。

## 歌词来源优先级

不同来源下，歌词的取用逻辑不同。

### 在线歌曲

- 优先从在线接口读取歌词
- 支持逐行、逐字、翻译歌词

### 本地歌曲

默认优先级通常是：

1. 同目录歌词文件
2. 音频内嵌歌词
3. 在线匹配歌词

你可以在右侧“本地”面板手动切换使用哪一种歌词来源。

### 歌词自动匹配与源优先级
针对在线与本地歌曲，Folia 会在匹配歌词时按照内置算法对歌词进行打分。打分机制会综合考量：是否逐字、是否存在副歌信息等。如果未能找到完美匹配，你也可以通过界面**手动指定优先的歌词来源（网易云、QQ音乐、酷狗等）**，匹配窗口内的结果会按照智能得分自动排序，方便你选取最优版本。

### Navidrome 歌曲

- 优先使用 Navidrome 返回的歌词
- 也可以手动切换到在线匹配歌词

## 歌词格式支持

项目代码中已经支持或识别以下格式：

- `lrc`
- `enhanced-lrc`
- `yrc`
- `qrc`
- `vtt`
- 常见音频标签中的内嵌歌词

这意味着同一首歌可以来自不同来源，但最终仍会被整理成统一的时间轴歌词数据。

## 适合录屏或展示的设置

如果你想用 Folia 做展示、录屏或舞台背景，建议：

- 先选定视觉模式
- 生成 AI 主题或手动设置配色
- 在实验室设置中隐藏不需要的 UI
- 使用桌面版的遥控窗进行控制
- 使用桌面版的视频导出功能录制

此外还可以使用 stage API 来开发外部控制程序，实现编排。
